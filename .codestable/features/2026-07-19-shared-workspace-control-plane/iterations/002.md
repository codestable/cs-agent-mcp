# Iteration 002：共享控制面主路径

## 当前理解

S0 已证明 Workspace 边界提取等价。本轮按批准设计实现 S1-S5 主路径，重点是唯一 Broker/Facade
owner、有状态 transport bridge、共享 Workspace registry 与 lease/grace 生命周期。

## 实现方式

先用可证伪测试覆盖并发启动、认证、standalone GET SSE、roots reverse-ready、共享与隔离，再实现
Broker 和 registry。关键共享、SIGKILL 与 lock 断言使用真实多进程 E2E，不以 mock 替代。

## 本轮变更

- 新增 Broker protocol、discovery、bridge、server 与 registry。
- 根 session 使用 Broker credential；managed loopback 保留 Facade bearer identity。
- roots 等待 initialized 与按 session 隔离的 reverse-ready，并增加独立有界错误。
- 同 Workspace 合并初始化并共享 root actor、Facade/runtime 与 Agent 树；不同 Workspace 隔离。
- 加入 lease、heartbeat、grace、Broker idle 与 crash/session 恢复主路径。

## 验证证据

- `pnpm run check`：236/236 通过。
- 双 client 交叉 list/status/wait/cancel/events/create/destroy、A 退出后 B 继续、多 Workspace 隔离通过。
- GET 405、SSE disconnect、5 秒 never-ready timeout 与 session id 隔离通过。
- Broker/frontend SIGKILL、credential 脱敏与 persistent session 恢复主路径通过。

## 遇到的问题

首次独立代码审查没有放行，指出旧协议 replacement 的 active-session 保护、pending initialization
计数、Workspace 关闭顺序/错误传播和部分真实竞态证据仍需加固。

## 下一次尝试

逐项闭合审查 finding，并重跑全量与多进程证据；不扩大公开契约。

S1-S3 完成，S4-S5 主路径完成但保留 review/fixing；Goal 继续 active。
