# Iteration 003：生命周期与恢复加固

## 当前理解

主路径已工作，但 production 并发正确性还取决于旧 owner replacement、零 lease 活跃 session、关闭
失败传播与进程锁恢复。必须在进入 QA 前把这些路径变成真实可重复证据。

## 实现方式

针对审查 finding 增加对抗测试后窄修复：旧 Broker 延迟释放真实锁、pending root session、ordered
shutdown、closed gate 和 persistent resume 均直接验证行为与 token/generation。

## 本轮变更

- 升级判断计入 zero-lease active session；replacement 等待旧 owner 释放真实 `broker.lock` 后重试。
- 未完成初始化的 root session 参与活跃判断，前端 SIGKILL 后可加入同一次初始化。
- Workspace 关闭改为 HTTP stop/close、Facade/runtime shutdown、最后释放 lock。
- Facade/runtime 增加 closed gate，关闭失败以 `AggregateError` 上抛。
- 补强 Broker crash 后 persistent session resume/fail-closed 证据。

## 验证证据

- `pnpm run check`：242/242 通过。
- 活跃旧协议 owner 不被关闭，inactive owner 延迟释放真实锁后 replacement 通过。
- 慢 Workspace 初始化时 frontend SIGKILL，replacement 加入同一初始化通过。
- Broker SIGKILL 后 session 真正 resume；缺失 session 返回 `SESSION_RESUME_REQUIRED`。

## 遇到的问题

真实 Codex/Claude C16 再次受外部环境阻断。后续审查还要求跨时区 process generation、并发 stale
compare/delete 竞态，以及 roots 延迟超过 grace 时 lock token 不变的更强证据。

## 下一次尝试

增加跨时区双进程和双 stale contender 对抗测试；让 unresolved roots 暂停 grace，并以真实 stdio
roots handler 延迟跨越 grace。随后重跑全量 review、QA 与功能验收。
