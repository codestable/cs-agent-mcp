# Iteration 008：Top Attach 完整会话与最终收口

## 当前理解

owner 明确 Top Attach 的预期是像 Codex/Claude 一样阅读已有完整上下文，而不是 Facade 事件列表；
runtime 已持久化原生 session record，因此 TUI 不应维护第二份历史。

## 实现方式

diagnostics 以确定性的 `mcp-<rootExecutionId>-<agentId>` 映射只读现有 session record，将 User、Agent、
Thinking、ToolUse、ToolResult、Resume、mention 与媒体元信息投影为 conversation DTO。Top controller
每秒 single-flight 刷新该 DTO；Facade attach stream 只更新 header 与 terminal reason，独立
`agents attach --json` 保持原事件流契约。

## 本轮变更

- Top Attach 改为完整只读会话视图，支持多行换行、实际渲染行滚动、paused/live 与未读计数。
- redacted thinking 只投影固定占位，不携带原始 payload；工具名限宽，正文保持可见。
- conversation 与 Facade event 类型分离，慢刷新合并为 single-flight 并由 generation 防止过期提交。
- 退出和返回列表不再等待不可取消的 session read；终端先恢复，captured refs 防止旧 pump 清空新
  Attach。
- README、架构文档、CHANGELOG、TUI PTY fixture、checklist、review、QA 与验收证据同步。
